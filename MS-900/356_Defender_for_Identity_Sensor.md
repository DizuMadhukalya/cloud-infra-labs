# Defender for Identity Sensor

The Defender for Identity Sensor is installed on Domain Controllers to monitor network traffic and authentication activities.

It collects security signals and sends them to Microsoft Defender for analysis.

---

Prevent breaches

It detects suspicious activities early to prevent attacks.

Example:

Detects brute force login attempts.

---

Detect threats

It identifies:

- Pass-the-hash attacks
- Credential theft
- Privilege escalation

Example:

If attacker uses stolen credentials to access multiple systems, sensor detects abnormal authentication.

---

Investigate incidents

It provides detailed investigation timeline.

Example:

Shows attacker path from compromised user account to domain admin access.

---

Respond to threats

Admins can:

- Disable compromised accounts
- Reset passwords
- Isolate systems

---

Conclusion:

Sensors provide deep visibility into identity threats.
