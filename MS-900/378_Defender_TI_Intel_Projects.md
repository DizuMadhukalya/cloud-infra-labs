# Commit 378 - Intel Projects in Microsoft Defender Threat Intelligence

## Overview

Intel Projects in Microsoft Defender Threat Intelligence allow security teams to organize, track, and manage threat intelligence investigations in a structured and collaborative manner. These projects serve as dedicated workspaces where analysts can collect indicators, monitor threats, analyze adversary infrastructure, and coordinate response efforts.

Intel Projects help security teams focus on specific threats such as ransomware campaigns, phishing attacks, or Advanced Persistent Threat (APT) groups.

---

## Key Features of Intel Projects

### 1. Centralized Threat Investigation

Security analysts can collect and store:

- Malicious IP addresses
- Suspicious domains
- File hashes
- Threat actor profiles

This provides a centralized place to monitor and analyze threats.

---

### 2. Indicator Tracking

Analysts can monitor indicators for changes, such as:

- IP address ownership changes
- Domain registration updates
- Infrastructure reuse by attackers

---

### 3. Collaboration Across Security Teams

Multiple analysts can work on the same project, share findings, and coordinate response.

---

## Real-Life Use Case Example

A financial organization detects phishing emails targeting employees. The security team creates an Intel Project to track:

- Malicious sender domains
- Command and control IP addresses
- Related phishing infrastructure

Using this project, they block malicious domains across firewalls and prevent credential theft.

---

## Benefits

- Organized threat investigations
- Improved collaboration
- Faster incident response
- Better threat visibility

---

## Conclusion

Intel Projects enable structured threat intelligence analysis and improve an organization's ability to detect and respond to cyber threats efficiently.
