# Configuration of Service, Insights, and Signals in Defender for Identity

Defender for Identity requires proper configuration to collect and analyze identity signals.

---

Service configuration includes:

- Connecting Active Directory
- Installing sensors
- Enabling data collection

---

Insights provided include:

- User risk scores
- Attack paths
- Identity exposure analysis

---

Signals include:

- Login activities
- Authentication failures
- Privilege changes

---

Real-life example:

If admin privileges are assigned unexpectedly, Defender generates an alert.

---

Conclusion:

Proper configuration ensures accurate threat detection.
